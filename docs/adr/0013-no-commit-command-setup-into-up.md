# The CLI wraps tools, not git: no `commit` command, `setup` folded into `up`

**Status:** accepted

Two surface cuts made after field-testing the freshly built CLI, both trading a command away for less cognitive load. **(1) `dobby commit` was removed.** Wrapping git (auto-stage, formatter-retry, push, `--pr`) invited edge cases for little value over what git/gh already do well. The cross-project standard is a CONVENTION instead: **`bunx dobby check --fix` is the pre-commit gate** — whoever commits (a human, the `/dobby:commit` skill, CI) runs it first; `--fix` applies Biome's safe fixes project-wide so the gate never fails on formatting the edit hook didn't reach. The `/dobby:commit` skill keeps the judgment (doc-sync, message, PR body) and performs the git/gh ceremony directly. Git hooks stay dead in consumers — the gate lives in the convention, not in `.git/hooks`. **(2) `dobby setup` was merged into `dobby up`.** The user always wants a workspace running when opening it, so a separate prepare step was pure ceremony: `up` now runs the setup phase (`bun install` + `.worktreeinclude` re-materialization + config `setup[]` extras, fail-fast) and then the run phase, idempotently; a no-app project stops cleanly after setup ("no app to run"). Both removed names hit the ordinary unknown-command path — no tombstones, no aliases.
